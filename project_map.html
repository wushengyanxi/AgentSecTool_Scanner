02

一句话架构：一条流水线

数据从发现流到落地：发现候选 → 只读探测判定 + 取版本 → 落 SQLite。发现有两路（自己扫的"主动" + 查 FOFA 的"被动"）；探测器对候选做确认与取版本（无论候选哪来的，探测都是自己做）。左侧色条 = 完成度。

绿条＝真实数据/单测验证过；橙条＝能跑但有桩或缺陷；灰虚线＝未做。越往下游越空——发现与探测扎实，"把版本对到漏洞、汇成面板、驱动披露"几乎为零。

03

判定思路：怎么确认一台是 OpenClaw

核心：不靠单条特征，靠多条"跨协议表面"的信号联合打分。OpenClaw 网关在三个互相独立的表面上留痕：

  • WebSocket 协议面：连上 ws://h:18789/，服务端立即推一帧 connect.challenge 事件——这是最强单证（几乎无他者有此行为）。
  • HTTP 静态资源面：首页 <title>OpenClaw Control</title>、favicon.ico 的 md5（f58854f6…）。
  • HTTP 路由/头面：/healthz 回 {"ok":true,"status":"live"}、/__openclaw/control-ui-config.json 回 401 + 特征 CSP、响应头三件套（X-Content-Type-Options/Referrer-Policy/Permissions-Policy）。

判定规则："命中 WS 挑战（强单证）或 ≥2 个不同表面命中"。置信度用 noisy-OR 跨表面合成。为什么强调"跨表面"——同一表面的信号（如 WS 挑战 + 未授权拒绝，或 favicon + 标题）伪造一个就能伪造另一个；只有跨表面（WS × HTTP静态 × HTTP路由）才真正抗伪造。判别力/置信度的完整论证见 openclaw-fingerprint-phase0-and-stack.html §01。

全程只读、良性：探测只发"HTTP 升级 + 无鉴权 GET"，绝不发 connect 帧或 config.apply（后者本身是 CVE 利用面）。这条以单测固化（probe_test.go 断言发送字节里没有越界帧）。

代码位置
openclaw/ws.go · probeWS() — WS 升级 + 读首帧，只读
openclaw/http.go · probeHTTP() — 四个 GET，抽取标题/favicon/healthz/CSP/头
openclaw/signals.go · score() — 跨表面打分 + 判定（noisyOR）
openclaw/probe.go · Probe() — 编排上面三步，产出 Result
openclaw/result.go · Evidence/Result — 信号与结果字段

已验证 对本地靶机 + 真实中国实例 confidence=1.0、六信号全中。坑：置信数字是工程先验、未按真值语料校准 中；HTTP-only 路径下克隆页可能误报（WS 挑战可破，但被反代时拿不到）低。

04

取版本思路：直读 + 隐式反推

版本既不在标题也不在 favicon 里。favicon.ico 自 v2026.1.8 加入后从未变过（实测仓库该文件仅一次提交），所以它跨版本恒定、只能判定 OpenClaw、不能区分版本；又因页面声明的主图标是 favicon.svg（.ico 只是 /favicon.ico 兜底），FOFA 的 icon_hash 只覆盖一小部分实例。取版本两条路，按优先级：

  • 直读（high 置信）：GET /__openclaw/control-ui-config.json 若回 200，体里有 serverVersion 字段——直接是精确版本。仅疏于防护（auth=none）的实例可得；已防护实例此接口 401。
  • 隐式反推（中置信）：401 时，用主机无需鉴权可观测、且随版本变化的量去指纹库匹配——主信号是首页引用的 assets/index-<hash>.js/.css（Vite 内容哈希，同版本恒定、跨版本不同），备用是 CSP 的 sha256。

指纹库怎么来：harness/build_corpus.py 在 Docker 里逐个起已知版本的 OpenClaw，用 ocprobe 记录其资产哈希/CSP/favicon，写进 fingerprints/fingerprints.json。匹配时资产哈希集合精确匹配优先。

代码位置
openclaw/version.go · determineVersion() — 先直读 serverVersion，否则走隐式
openclaw/fingerprints.go · MatchImplicit() / LoadFingerprintDB() — 资产哈希/CSP 匹配版本
openclaw/http.go · probeHTTP() — 从首页正则抽 assets/index-<hash>、算 CSP 哈希、解析 serverVersion
fingerprints/fingerprints.json — 指纹库（version → 资产哈希/csp/favicon）
harness/build_corpus.py · record_signature() — 逐版本起容器采集签名入库

部分 机制完整、单测过、真实实例直读出 2026.3.x。坑（高 高）：指纹库只灌了 1 版（2026.5.17）→ 跑其他版本的已防护实例资产哈希对不上任何条目 → 版本"未知"。 解法：用 harness 把 2026.1.x~2026.4.x 等灌进库。

05

发现思路：怎么找到候选（主动 + 被动）

发现只回答"哪些 IP:port 值得探测"。两路并取：

主动（自己发包扫 IP 段）：三后端——masscan/ZMap 无状态 SYN 扫端口集（全网级，需 root），internal 内置 async connect 扫描（无 root 兜底，/16 上限）。配黑名单（默认排除保留/私有段）。这条路是"全网能力"的来源，也是进度块队列（07）要驱动的对象。

被动 FOFA（查 FOFA 索引，不向目标发包）：这是中国大陆普查的主路。关键认识：FOFA 只给候选、不给版本、也不等于确认（它不做 WS 握手）；候选确认 + 版本仍由我们探测器做。默认候选查询（实测 country=CN）：(title="OpenClaw" || icon_hash="-805544463")——title 用宽匹配（FOFA 的 = 即"包含"）以多召回改名/汉化实例，比精确短语 "OpenClaw Control" 多约 3 千，误报交给逐台探测过滤。可叠加 app="OpenClaw"（FOFA 产品指纹，单独即 ≈9.9 万，与 title∪icon 高度重叠）。查询不写死：CLI --query 可传完整 FOFA 语句覆盖默认。

FOFA 额度策略（绑定约束 = 10 万条数据/月，查询次数不紧张）：额度由 FOFA 服务端强制（超了即拒），本地只用 --max-records 封顶 + 耗尽时优雅停续拉；版本新鲜度靠自有探测器复扫候选（不耗 FOFA 额度）。普查切分用时间窗，不靠游标跨月：--before/--after 是 FOFA 操作符（过滤 lastupdatetime），切出确定性、可重复的块——先拉新鲜窗（after，更可能在线），陈旧窗按月低优先补。fofa_state 里的 Search After 游标只管一次运行内的断点续（重启可续，且按查询语句校验）；跨月/跨设备靠"重跑同一窗口查询 + 按 (ip,port) 去重累积"，不依赖游标长期有效。

07

进度机制：主动扫描怎么断点续、跨设备分

这套是为主动全网扫描准备的（被动 FOFA 不需要它，翻页即可）。思路：把"待扫范围"切成 CIDR 块（不是逐 IP——中国大陆 3.43 亿地址，逐 IP 记录不现实），按战役（campaign）组织（暴露面随时间变，要能定期重扫），块状态 pending→in_progress→done，原子领取、断点续、超时回收。

"待扫范围"全集 = scope/cn_ranges.py 从 APNIC 生成的中国大陆 CIDR（实测 5489 段 / 3.43 亿地址，切 /16 块共 9713 个）。跨设备：同一 SQLite 文件只支持同机多进程；真要多设备，要么 masscan/ZMap 分片（--shard，无需共享状态），要么把状态库换成共享 DB。

09

怎么跑（凭据走 fofa/fofa.ini 或环境变量，不硬编码）

  • 测试全过：make test（go + 24 个 python 单测）。
  • 本地全链 demo：先起靶机容器，再 make demo（发现 localhost → 探测 → 落地 → 统计）。
  • FOFA 普查：先 cp fofa/fofa.example.ini fofa/fofa.ini 填凭据；在项目根目录（非 fofa/ 内，否则 No module named fofa）跑 make fofa-info（额度）→ python3 -m fofa pull --full --before 2026-05-30（时间窗全量，带进度，可续）→ make fofa-provinces → make fofa-export → prober/bin/ocprobe -f candidates.csv --fingerprints fingerprints/fingerprints.json -o results.jsonl → make load → make fofa-pv（按省×版本）。
  • 生成中国范围 / 进度：python3 -m scope.cn_ranges → python3 -m progress seed --campaign cn-2026w24 --cidrs scope/cn-cidrs.txt。
  • 扩充指纹库：python3 harness/build_corpus.py --image <openclaw 镜像> --version <版本>。

凭据：FOFA 走 fofa/fofa.ini（[fofa] email/key，已 gitignore）或环境变量 FOFA_EMAIL/FOFA_KEY（优先级：入参 > 环境变量 > 配置文件）；Shodan 走 SHODAN_API_KEY。绝不硬编码、不入库。真扫公网属运营动作，须在授权扫描主机上限速、可识别地跑。

新接手的人记住三句：前半段（发现 + 探测 + 取版本机制 + FOFA 普查）已扎实并在真实中国数据上跑通；后半段（版本→CVE、面板、披露）几乎为零；中间埋着两个会让数据出错的坑——指纹库仅 1 版、资产去重碰撞。 想动手，建议先修资产去重（让数据可信）、再灌多版本指纹库、再做版本→CVE。每个功能对应哪段代码，见上面各节的"代码位置"。
