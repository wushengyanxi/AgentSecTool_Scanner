AgentSecTool Scanner · 技术交付说明

OpenClaw 扫描器基于哪些服务特征研判，以及如何让单 IP 研判更精准

核心结论

当前扫描器对一台主机的研判，建立在 1 次 WebSocket 升级握手 + 4 条无鉴权 HTTP GET 共采集的 8 个服务特征 之上，按“跨协议表面组合”做置信合成；判定门槛为 WS 命中 || ≥2 个表面命中 || 置信≥0.97。版本走两条路：疏防实例直读 serverVersion，已防实例靠资产哈希反推。版本指纹库已覆盖自更名起的全部 70 个正式版（共 71 条），30 个随机版本经真实容器端到端验证全部识别正确。其中 6 组相邻版本因前端 bundle 未变而资产指纹相同，反推时如实给出版本区间；尚未消费的信号（TLS 证书、WS 载荷结构、未授权高危态）是进一步精准化的方向。

01

扫描器现在“基于什么”——一次探测的动作序列

对每个候选 IP[,port]（默认端口 TCP/18789），探测器执行一段固定的、严格只读的序列：先发一次 WebSocket 升级握手，再发四条无鉴权 GET，从各响应里提取证据，最后合成判定。除 WS 升级请求外不向目标写入任何内容，更不触碰任何会改变目标状态的帧——这一“只读不变量”由单元测试固化。

深色框为被测目标，仅作数据落点；五个白框是固定探测序列（橙线＝读取动作，唯一的写动作是 ① 的 WS 升级请求）；右侧灰框是把证据合成判定与版本的打分环节。动作序列与提取字段对应 prober/openclaw/ws.go、http.go。

03

判定与取版本的合成逻辑

八个特征归入三个相互独立的协议表面——WS 协议面、HTTP 静态面、HTTP 路由/头面。每个表面取该面内命中信号的最大概率，再用 noisy-OR 合成总置信度。判定有三个并列触发条件，任一满足即判为 OpenClaw。版本独立于判定，走优先级回退。逻辑全部 代码已证实，见 signals.go 与 version.go。

// 判定（signals.go:46-54）
confidence = noisyOR(pWS, pStatic, pRoute)   // 1 − ∏(1−p)
surfaces   = count(p > 0  for p in [pWS, pStatic, pRoute])
isOpenClaw = ev.WSChallenge || surfaces >= 2 || confidence >= 0.97

// 取版本（version.go:5-9，优先级回退）
if ev.ServerVersion != ""      return normalize(ServerVersion), "direct"   // 直读，需 auth=none
else                            return MatchImplicit(ev)                       // 反推，需指纹库
//   MatchImplicit：资产哈希集合精确相等 ▸ 退到 CSP-SHA256 精确相等 ▸ 否则版本为空
  • WS 命中可单独定案。ev.WSChallenge 为真时 isOpenClaw 直接为真——connect.challenge 是协议级信号，代理／WAF 难以伪造，故作最强单证。
  • 无 WS 时需 ≥2 个表面交叉。例如「favicon ∧ /healthz」或「title ∧ control-ui-config」，用跨表面冗余压低误报。
  • 版本直读的前提是疏防。仅当 control-ui-config 返回 200（即 auth=none）才读得到 serverVersion；一旦返回 401，直读失效，只能反推。
  • 常量 detectConfidenceFloor=0.90 已定义但未接入（consts.go:27）；判定实际用的阈值是硬编码的 0.97。属代码与注释的轻微不一致，不影响行为，记录在案。
04

制约单 IP 研判精准度的缺口

以下缺口均基于源码事实，是“为什么对一台主机研判还不够精准”的真实原因，而非泛论。按对单 IP 研判的影响程度排序。

缺口
说明（基于源码）
6 组相邻版本不可区分 代码已证实
指纹库已覆盖全部 70 个正式版（原“仅 1 版”的硬缺口已解决）。残留限制是：纯后端改动的相邻版本前端 bundle 不变、资产指纹相同，实测有 6 组共 12 个版本无法区分（1.12=1.13、1.23=1.24、3.7=3.8、4.7=4.8、4.20=4.21、5.5=5.6）。被 401 挡住 serverVersion 又落在这些组的实例，只能反推到“两版之一”，探测器对此返回 implicit-range 候选集而非谎报单值。
多数实例无法直读版本 代码已证实
直读 serverVersion 需 control-ui-config 返回 200 且响应含该字段。实测：该端点自 v2026.4.22 起加鉴权（无 token 即 401），serverVersion 字段约自 v2026.3.12 才加入。现实公网实例多为新版（返回 401），多数只能靠资产指纹反推，能直读精确版本的目标占比很低——这正是全量指纹库的价值所在。
WS 载荷只做字符串匹配 代码已证实
ws.go:66 仅 strings.Contains 判断帧里是否含 "event":"connect.challenge"，不解析 JSON、不校验 type、不提取 nonce/ts/payload。结构化信息（含可能的协议代次线索）被丢弃。
TLS 证书完全未采集 代码已证实
HTTP/WS 客户端均 InsecureSkipVerify=true（http.go:80），握手后未提取证书 CN/SAN/颁发者/自签状态/指纹。一个可独立佐证、且对资产聚类去重很有价值的表面被整体放弃。
“未授权可访问”未列为一等输出 代码已证实
control-ui-config 返回 200（auth=none）即意味着配置可被任意读取，这是高危事实，但当前只并入置信打分，未在结果里单列 unauthenticated 标记。
资产哈希要求集合精确相等 代码已证实
fingerprints.go:40 用 setsEqual。若某版本只改 js 未改 css，或 CDN 重写了文件名，集合不等即整体不命中，无部分匹配/相似度兜底。
无暴露时效信息 代码已证实
单 IP 探测不记录、也不利用 lastupdatetime 类时效，无法区分“当前在线”与“历史残留”。
05

要补充哪些信息，单 IP 研判才更精准

下列改进直接针对上一节的缺口。每条说明“补什么、怎么补、补完哪个信号变强”。可信级别已逐条标注：标 代码已证实 的是基于现有代码即可扩展的工作，标 规划 · 未实现 的是源码尚无、需新建的能力。所有手段都必须维持“只读、非侵入”的不变量。

① 把指纹库扩到全量 已完成

已用 harness/collect_corpus.py 采全自更名起的 70 个正式版（共 71 条）。主路径从官方镜像（ghcr.io/openclaw/openclaw）静态层 docker cp 出 dist/control-ui，不起服务、逐字节等同分发（57 版）；无官方镜像的最老版本回退到一次性 node:22 容器内 pnpm 构建，宿主机零污染（13 版）。同时改 MatchImplicit 返回候选集、新增 implicit-range 来源与 version_candidates 字段，命中不可区分区间时如实报候选。

→ 30 个随机版本经真实容器端到端验证全部识别正确（10 直读 / 17 反推单值 / 3 区间）。被 401 挡住版本号的实例现在也能反推出精确版本或版本区间。

② 解析 WS connect.challenge 的 JSON 载荷 规划 · 未实现

把 ws.go:66 的 strings.Contains 升级为 json.Unmarshal，校验 {"type":"event","event":"connect.challenge","payload":{…}} 完整结构，并提取握手字段。

→ WS 信号从“字符串巧合可伪造”升级为结构化确认；若载荷携带协议版本号，还能为版本判定增加一个不依赖 HTTP 的来源。

③ 采集并研判 TLS 证书 规划 · 未实现

对 TLS 目标，在握手后从 tls.ConnectionState 提取证书 CN/SAN/颁发者/自签状态/有效期/指纹，写入 Evidence，并视情况纳入指纹库。

→ 新增一个独立表面提升 noisy-OR 置信；佐证服务归属与是否自部署；为按证书去重、资产聚类提供比 IP 更稳的身份键。

④ 把“未授权可访问”提升为一等研判输出 规划 · 未实现

当 control-ui-config 返回 200（auth=none）时，在结果结构里单列 unauthenticated=true 的高危标记，而不仅并入置信分。

→ 让“这台配置可被任意读取”这一风险结论显式可消费，直接服务于风险分级与协同披露的优先级排序。

⑤ 为资产哈希反推加部分匹配兜底 规划 · 未实现

当资产集合不完全相等时，按 js/css 命中比例给出候选版本与一个置信度，而非直接判空；并对 CSP 哈希前做规范化（指令排序、去多余空白）。

→ 缓解 CDN 改名、部分升级导致的整体丢失，提升反推的召回与稳定性。

⑥ 保留完整响应头与原始证据快照 规划 · 未实现

除现有字段外，留存 Server 头与完整响应头快照、首页 HTML 结构特征、control-ui-config 的 JSON 键集合。

→ 既新增弱佐证信号，也为未来发现新版本特有指纹留存原始材料，使版本指纹可在资产哈希被 CDN 抹掉时仍有补充维度。
