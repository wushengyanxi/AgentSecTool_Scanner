AgentSecTool Scanner — 项目文档

在授权范围内发现公网暴露的 OpenClaw 网关，只读地判定其是否为 OpenClaw、测定版本并归档，服务于后续的漏洞协同披露。本文档是工程接手契约：让接手者无需询问原作者即可理解目标、跑通最小闭环、定位模块、解释规则、执行测试、判断边界、排查失败。

一句话定义
OpenClaw 公网暴露面扫描器：发现 → 只读探测判定 → 版本测定 → SQLite 归档
当前状态
主链路已对真实数据跑通（assets 5762 / observations 5762 / probe_records 28810），核心研判有 Go 单测固化
技术栈
Go 1.24（探测引擎 ocprobe）+ Python 标准库（发现 / 测绘 / 落地 / 看板）+ SQLite + echarts（看板地图）

data/scanner/scan_results.sqlite（扫描结果，101MB，已 gitignore）

核心能力

  • 只读探测判定：对单个 IP:端口 做 WebSocket + HTTP 只读探测，按证据组合白名单（C1/C2/C3）二元研判是否为 OpenClaw。prober/openclaw/probe.go:22-74, signals.go:26-56
  • 版本测定：优先直读 serverVersion（direct），否则用首页构建产物的资产哈希集合精确匹配指纹库反推（implicit / implicit-range）。version.go:13-23, fingerprints.go:41-56
  • 发现层：主动扫描（masscan / zmap / 内置 async connect）+ 被动种子（FOFA / Shodan），合并去重产出 candidates.csv。discovery/
  • 落地归档：results.jsonl → SQLite，入库时按 IP 富化地理位置。store/load.py
  • 看板：纯标准库只读看板，含可切换世界/中国的 2D 散点地图。dashboard/

核心约束

  • 探测全程只读（函数级有单测固化）：探测只发 HTTP 升级请求与无鉴权 GET，绝不发 WS 数据帧或利用面请求。ws.go:24-79, probe_test.go:55-89 注：整条 runner 调用链未抓包断言，见 验证。
  • 凭据绝不入库：FOFA / MaxMind 凭据走 .ini 或环境变量，配置文件已 gitignore。
  • 真实数据绝不入库：data/、scan_test/results/ 含真实暴露主机数据，已 gitignore。

非目标

  • 不做漏洞利用——只读探测，绝不发 connect 帧或 config.apply（后者是 CVE 利用面）。
  • 不把 FOFA / ClawSec 的平台标注当作项目确认事实——它们是外部测绘数据源、只提供候选与平台自己的版本标注。
  • 不处理 IPv6（ocprobe 通配枚举只对 IPv4 第三位有意义）。
  • 下游的「版本→CVE 映射、协同披露流程」尚未实现，见 已知限制。

快速开始 Quickstart

最小可复现闭环。所有命令在项目根目录（AgentSecTool_Scanner/）运行——Python 模块以 python3 -m 调用，Makefile 已设 PYTHONPATH；在子目录内运行会报 No module named …。

WARNING · 不要用 make demo / make probe 作为最小路径

架构 Architecture

系统分两条链路。正式主链路：发现 → 探测 → 落地 → 看板。供料与辅助链路：FOFA / ClawSec（被动测绘数据源）、geoip（富化）、指纹库、progress / scope（调度）；fingerprint / scan_test 是一次性历史工具。判定与版本测定只发生在 ocprobe。

全灰度。实线＝正式主链路数据流；虚线框与虚线箭头＝供料 / 辅助链路（可选喂入）。粗边框的 ocprobe 是唯一做判定与版本测定的组件——强调靠边框粗细而非填充色。FOFA / ClawSec 只提供候选与平台标注，需经 ocprobe 才成为项目确认事实。箭头方向＝数据流向。

测试项 T1–T7（探测采集的证据信号）

每次探测对一台主机采集 7 个测试项，按可仿冒难度分确证 / 强 / 弱三级。定义见 result.go:5-13，采集见 http.go / ws.go。下表每项的「真实返回样字」取自本项目对 6-20 中国 18789 段的实测结果（scan_test/results/cn-2026-06-20.jsonl，约 1.2 万个确认实例），非杜撰。

T1 确证 control-ui-config 返回 200 且体含 serverVersion

采集方式
GET /__openclaw/control-ui-config.json，解析 JSON 取 serverVersion http.go:59-80
真实返回样字
状态行 HTTP/1.1 200，响应体 JSON 含 "serverVersion":"2026.4.11"（实测版本字样：2026.4.11 / 2026.3.11 / 2026.3.12 / 2026.3.7 / 2026.3.24 等，共 5985 例直读到版本）
这意味着什么
该路径是 OpenClaw 控制台前端的配置端点，返回体里的 serverVersion 是服务端自报的运行版本号。能拿到这个字段，说明背后跑的就是 OpenClaw 的控制面，且这台没对该端点做鉴权防护。
与判定的直接联系
确证级、单独达标（C1）。仿冒一个返回「200 + 合法 serverVersion JSON」的该专有路径，成本等同真实现该端点契约。版本也由此直读（source=direct）。
不应如何理解
不是「该路径返回 200」就命中——必须体里真有 serverVersion 字段。200 但无该字段不算 T1（走 C3 或不命中）。

T2 强 WebSocket 连上后服务端立即推 connect.challenge 帧

采集方式
WS 升级后只读首帧，匹配事件名 ws.go:24-79（只读不变量：绝不回发任何帧）
真实返回样字
先 HTTP/1.1 101 Switching Protocols，随即推一帧：{"type":"event","event":"connect.challenge","payload":{"nonce":"643e64e5-…","ts":1782139659263}}
这意味着什么
客户端一连上，服务端主动下发一个带 nonce/时间戳的握手挑战——这是 OpenClaw 网关 WS 协议的固有开场行为，是动态协议交互，不是某个能被下载保存的静态文件。
与判定的直接联系
强证据，是 C2 / C3 的必要条件。要伪造它必须真跑一个会主动推 connect.challenge 的 WS 服务端，他者框架罕有此行为。但单独 T2 不达标（防单信号被仿冒）。
不应如何理解
不是「能建立 WS 连接」就命中——必须收到 connect.challenge 事件帧。

T3 强 control-ui-config 返回 401

采集方式
同 T1 的 GET，看状态码 ==401 http.go:79
真实返回样字
状态行 HTTP/1.1 401，响应体 {"error":{"message":"Unauthorized","type":"unauthorized"}}（实测 control-ui 状态分布：200 占 9653、401 占 2604、404 占 1299）
这意味着什么
这台 OpenClaw 对配置端点做了鉴权防护——端点存在（所以不是 404）但要求授权（所以是 401）。那段固定的 unauthorized JSON 体本身就是 OpenClaw 的特征错误响应。
与判定的直接联系
强证据，与 T2 联合构成 C2。它专门覆盖「已防护、拿不到版本号」的实例——这类实例 T1 不中，靠 T2+T3 仍能确认是 OpenClaw（只是版本未知）。
不应如何理解
401 才算 T3；返回 200 不算（200 走 T1 或不命中），404 说明端点不存在、更不算。

T4 强 /healthz 返回特征体

采集方式
GET /healthz，精确匹配体内容 http.go
真实返回样字
HTTP/1.1 200，体 {"ok":true,"status":"live"}（实测 healthz_match=true 占 10555 / 命中率高）
这意味着什么
OpenClaw 网关的健康检查端点返回的固定 JSON。这个 {"ok":true,"status":"live"} 字面体是 OpenClaw 的特征，区别于其它服务常见的 OK / {"status":"healthy"} 等。
与判定的直接联系
强证据，与 T2 联合构成 C2（当 T3 未中但 healthz 命中时）。是已防护实例的另一条确认路径。
不应如何理解
不是「/healthz 返回 200」就命中——必须体精确等于该特征 JSON。

T5 弱 favicon.ico 的 MD5 等于已知值

采集方式
GET /favicon.ico，算 MD5 比对常量 http.go:82-89
真实返回样字
MD5 f58854f6450618729679ad33622bebaf（实测 12181 例为此值、跨版本恒定；另有极少数离群值如 eb84864… 各 1 例，属自定义图标）

T6 弱 首页 <title> 含 OpenClaw 标记

采集方式
GET /，正则抽 <title> http.go:42-57
真实返回样字
OpenClaw Control（实测 12124 例）；离群样字：Clawdbot Control（62，疑似改名分支）、OpenClaw 控制台、甚至 军机处 · 三省六部总控台 / 中创云服智能体 控制（各 1，自定义标题）

不应如何理解

T7 弱 响应头三件套

采集方式
检查首页响应头是否同时含三个头 http.go:166-174 hasHeaderTriplet()
真实返回样字
三头需同时满足：X-Content-Type-Options: nosniff、Referrer-Policy: no-referrer、Permissions-Policy: …microphone=(self)…（实测 header_triplet=true 占 3239，命中率明显低于其它信号）
这意味着什么

不应如何理解
三件套齐全不单独证明是 OpenClaw；不齐也不否定（可能被反代改写了头）。

INVARIANT · 为什么分确证/强/弱三级

分级的依据是可仿冒成本，不是「看起来像不像」。确证（T1）与强证据（T2/T3/T4）要伪造必须真实现 OpenClaw 的专有端点契约或动态协议行为；弱证据（T5/T6/T7）都是可被静态复制或通用框架即可凑齐的表面特征。所以达标白名单只用确证级单独成立、或强证据跨表面联合，弱证据无论多少都不单独达标——见 判定规则。

资产哈希（assetHashMatch）

首页引用的 assets/index-<hash>.js / .css 文件名集合。哈希由 Vite 构建产物的内容算出，同版本恒定、跨版本不同。assetHashMatch = 该集合与指纹库某版本条目的 asset_hashes 精确相等（setsEqual，非子集）。它既是隐式版本反推的信号，也是判定规则 C3 的一个条件。fingerprints.go:41-56

WARNING · 资产哈希的采集面与 T5/T6 同源

资产哈希是从首页明文 HTML 正则抽取文件名得到的，扫描器不二次请求那两个资产、不校验真实字节。所以单独的资产哈希可被「镜像一份真首页」廉价伪造，与伪造 T6 同难度。因此 C3 要求它必须与 T2（动态 WS 行为）联合。详见 判定规则 C3。

版本来源 version_source

  • direct：直读 control-ui 的 serverVersion，最可靠。
  • implicit：资产哈希唯一命中指纹库某版本，反推得出。
  • implicit-range：资产哈希命中多个版本（构建产物相同、不可区分的版本区间，如 2026.3.7=2026.3.8），返回代表版本 + 全部候选。version.go:13-23

隐位 IP（masked_ip，ClawSec 专有）

ClawSec 平台公开列表把 IP 的第三位隐藏成 *（如 1.14.*.22）。补全靠 ocprobe 对该通配段枚举第三位 0–255。这是 ClawSec 数据特有，与本项目权威数据无关。

安全与运营边界 Security, Privacy, Operational Boundaries

授权边界

WARNING · 对公网的主动扫描属运营动作，须授权

ocprobe / masscan / zmap 会对目标 IP 发起真实连接。最小路径用本地靶机；大规模扫描须在获授权的主机上、对获授权的目标进行。本项目扫的 OpenClaw 实例多来自 ClawSec / FOFA 已公开列出的暴露实例。

默认允许 / 默认禁止行为（落到具体动作）

  • 允许（探测全程只读）：HTTP 升级请求、无鉴权 GET（/healthz、/、control-ui-config、/favicon.ico）、读 WS 首帧。
  • 禁止（有单测固化于函数级）：发任何 WS 数据帧、发 connect 帧或 config.apply（CVE 利用面）。ws.go 只 Write 一次升级请求；probe_test.go 断言不含越界帧
  • 可达性门槛：先一次 TCP 连接，连不上的目标直接返回、不发 HTTP/WS。probe.go

WARNING · 「整条链路只读」是函数级证据，非整链抓包

速率与限制

  • -rate（默认 50）：单 IP 每秒发包上限，但实现为 per-worker time.Sleep、非跨 goroutine 全局限速。同一 IP 多端口入队后各 worker 互不协调。HTML 不声称「严格全局单 IP 速率」。main.go minInterval
  • masscan 限速靠 --rate；blocklist 默认排除 16 个 RFC 保留/私有网段，非法 IP 一律排除。discovery/blocklist.py:6-30
