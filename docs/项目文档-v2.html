AgentSecTool Scanner：项目文档

AgentSecTool Scanner 在授权范围内发现公网暴露资产，调用内置或动态探测器观测目标实例、测定版本并归档项目事实。平台还接收上游漏洞需求，通过持续会话、容器 Harness 和人工审核派生可长期运行的项目测试能力。

平台主链路
发现 → 按资产类型探测 → 项目事实与版本归档 → 看板展示
能力生产链路
需求快照 → 智能体派生 → Docker Harness → 人工审核 → 动态能力仓
平台形态
平台入口同时加载内置 detector 和已审核容器 worker；OpenClaw 是内置分支
权威数据位置
store/data/scan_results.sqlite 与 derivation/data/derivation.sqlite

核心能力

  • 平台级探测调度：assetprobe --type <asset_type> 从内置注册表或动态能力索引选择 detector。prober/detectors/, prober/cmd/assetprobe/
  • 只读探测判定：每个 detector 对单个 IP:端口执行只读探测，输出统一的 asset_type / is_match / category 平台字段；OpenClaw 分支按 C1/C2/C3 二元研判。prober/openclaw/probe.go, signals.go
  • 版本测定：OpenClaw 分支优先直读 serverVersion（direct），否则用首页构建产物的资产哈希集合精确匹配指纹库反推（implicit / implicit-range）。version.go, fingerprints.go
  • 发现层：主动扫描（masscan / zmap / 内置 async connect）+ 被动种子（FOFA / Shodan），合并去重产出 candidates.csv。src/agentsectool_scanner/discovery/
  • 落地归档：results.jsonl → SQLite，按 asset_type:ip:port 去重，入库时按 IP 富化地理位置。src/agentsectool_scanner/store/load.py
  • 能力派生：需求包导入、任务级持续会话、验收项版本、本地 Docker Harness 和候选能力审核共同形成动态项目测试项。src/agentsectool_scanner/derivation/
  • 前端：资产看板提供时间窗口、资产类型、地图和通用事实报告；派生工作台承载需求导入、会话、验收、证据和审核。src/agentsectool_scanner/dashboard/

核心约束

  • 探测全程只读（函数级有单测固化）：探测只发 HTTP 升级请求与无鉴权 GET，绝不发 WS 数据帧或利用面请求。ws.go:24-79, probe_test.go:55-89 注：整条 runner 调用链未抓包断言，见 验证。
  • 凭据绝不入库：FOFA / MaxMind 凭据走 .ini 或环境变量，配置文件已 gitignore。
  • 能力必须验证后入仓：动态 worker 经过隔离 Harness、已确认验收项和人工审核后才进入运行索引。
  • 真实数据不进入版本库：各模块 data/ 和 output/ 目录可能包含暴露主机数据，相应路径已由 Git 忽略。

非目标

  • 不做漏洞利用——只读探测，绝不发 connect 帧或 config.apply（后者是 CVE 利用面）。
  • 不把 FOFA / ClawSec 的平台标注当作项目确认事实——它们是外部测绘数据源、只提供候选与平台自己的版本标注。
  • 不在平台层硬编码 OpenClaw 规则——OpenClaw 规则只属于 asset_type=openclaw 的 detector 分支。
  • 不处理 IPv6（ocprobe 通配枚举只对 IPv4 第三位有意义）。
  • 不由智能体直接发布能力——智能体只能形成候选，入仓由验证结果和操作人审核共同决定。

资产类型与探测器

平台层负责候选输入、detector 调度、统一结果字段、入库、查询和展示；具体资产类型的探测规则、证据采集、版本测定和报告话术由各自 detector 分支实现。OpenClaw 是当前内置的首个资产类型分支。

内置资产类型接入契约

  • 在 prober/detectors/<type>/ 实现 detector，并通过 detectors.Register() 注册唯一 asset_type。
  • 结果至少暴露平台字段：asset_type、detector、ip、port、is_match、category、matched、error_type、ts。
  • category 统一使用 confirmed、confirmed_no_version、suspect；探不到且无命中的记录不入库。
  • OpenClaw 的 is_openclaw、rule、C1/C2/C3、T1–T7 是兼容字段和专属规则，不应泄漏成平台通用规则。

漏洞需求驱动的项目测试项通过派生工作台接入。每个能力包包含一个独立项目测试项，同一 asset_type 可以登记多个能力包；运行索引以 capability_id 区分项目测试能力，并固定能力 manifest 与容器镜像 digest。

测绘能力派生

测绘能力派生链路把上游漏洞项目交付的扫描代码和说明文档转化为可持续执行的项目测试项。上游交付保持为独立目录；平台导入登记材料的不可变快照，并在本地派生数据库中管理会话、验收、attempt、Harness 证据、候选能力和审核状态。

需求包与补充包

每个需求目录必须包含 manifest.json、一个或多个 UTF-8 代码文件和一个 HTML、Markdown 或纯文本主文档。YAML、配置、日志和样例数据属于可选辅助文件，并由 manifest 的 auxiliary_files 登记。导入器校验安全相对路径、文件存在性和 SHA-256；未登记文件不进入智能体上下文。

{
  "schema_version": "1.0",
  "request_id": "CVE-EXAMPLE-project-x",
  "code_files": ["scanner.py"],
  "document_file": "report.html",
  "auxiliary_files": [
    {"path": "environment.yaml", "kind": "environment_hint"}
  ]
}

任务会话中的附件按消息形成独立补充包。补充包具有自己的文件清单和 hash，不覆盖原始需求快照。用户消息可以提问、补充约束、上传证据或要求智能体根据当前讨论继续执行；运行期间到达的新消息会排队进入同一任务会话。

派生、验收与审核

派生智能体读取代码和主文档，生成项目测试项、固定容器 worker、隔离样例环境、漏洞关联规则和展示模板。项目测试项在生产扫描中观测实例事实；验收项只约束候选能力在 Harness 样例中的行为。所有启用验收项必须经操作人确认，智能体不能修改已经确认的验收项。

阶段主要对象准入条件
需求导入原始快照、artifact 清单、任务会话manifest、路径和 SHA-256 校验通过
候选生成attempt、worker、环境、规则、展示模板候选文件只写入当前 attempt 工作区
容器验收Harness run、逐项结果、日志和 trace隔离环境中的启用验收项通过
候选审核能力 manifest、固定镜像 digest、来源证据验收项已确认，操作人批准入仓
生产执行能力索引、长生命周期 worker、项目事实运行时镜像 digest 与入仓记录一致

多测试项执行

一个能力包承载一个独立项目测试项，同一资产类型可以并存多个能力包。capability_packages/index.json 按 asset_type 保存能力数组；同一 capability_id 的新版本替换旧索引项，不影响同类的其他测试项。动态 detector 聚合各 worker 的状态、事实、证据、版本、漏洞规则和展示模板。冲突事实进入 _conflicts，依赖这些事实的漏洞适用性保持未知。

工作台入口

系统架构

系统包含扫描主链路、能力生产链路和供料辅助链路。扫描主链路为发现、平台调度、detector 探测、落地和看板；能力生产链路为需求快照、智能体派生、Docker Harness、人工审核和动态能力入仓；FOFA、ClawSec、GeoIP、指纹库、progress 与 scope 提供候选、富化和调度支持。项目判定与版本测定发生在内置 detector 或已入仓项目测试项中。

INVARIANT · 判定来源

FOFA / ClawSec / scan_test 都不做 OpenClaw 判定或版本测定。它们提供候选或平台自报的版本标注；是否真为 OpenClaw、项目认可的版本是多少，由 assetprobe --type openclaw 或 ocprobe 的只读探测得出。这是区分「项目确认事实」与「外部平台标注」的根本边界。

测试项 T1–T7（探测采集的证据信号）

每次探测对一台主机采集 7 个测试项，按可仿冒难度分确证 / 强 / 弱三级。定义见 result.go:5-13，采集见 http.go / ws.go。下表每项的「真实返回样字」取自本项目对 6-20 中国 18789 段的实测结果（tools/scan_test/output/results/cn-2026-06-20.jsonl，约 1.2 万个确认实例），非杜撰。

T1 确证 control-ui-config 返回 200 且体含 serverVersion

采集方式
GET /__openclaw/control-ui-config.json，解析 JSON 取 serverVersion http.go:59-80
真实返回样字
状态行 HTTP/1.1 200，响应体 JSON 含 "serverVersion":"2026.4.11"（实测版本字样：2026.4.11 / 2026.3.11 / 2026.3.12 / 2026.3.7 / 2026.3.24 等，共 5985 例直读到版本）
这意味着什么
该路径是 OpenClaw 控制台前端的配置端点，返回体里的 serverVersion 是服务端自报的运行版本号。能拿到这个字段，说明背后跑的就是 OpenClaw 的控制面，且这台没对该端点做鉴权防护。
与判定的直接联系
确证级、单独达标（C1）。仿冒一个返回「200 + 合法 serverVersion JSON」的该专有路径，成本等同真实现该端点契约。版本也由此直读（source=direct）。
不应如何理解
不是「该路径返回 200」就命中——必须体里真有 serverVersion 字段。200 但无该字段不算 T1（走 C3 或不命中）。

T2 强 WebSocket 连上后服务端立即推 connect.challenge 帧

采集方式
WS 升级后只读首帧，匹配事件名 ws.go:24-79（只读不变量：绝不回发任何帧）
真实返回样字
先 HTTP/1.1 101 Switching Protocols，随即推一帧：{"type":"event","event":"connect.challenge","payload":{"nonce":"643e64e5-…","ts":1782139659263}}
这意味着什么
客户端一连上，服务端主动下发一个带 nonce/时间戳的握手挑战——这是 OpenClaw 网关 WS 协议的固有开场行为，是动态协议交互，不是某个能被下载保存的静态文件。
与判定的直接联系

INVARIANT · 为什么分确证/强/弱三级

分级的依据是可仿冒成本，不是「看起来像不像」。确证（T1）与强证据（T2/T3/T4）要伪造必须真实现 OpenClaw 的专有端点契约或动态协议行为；弱证据（T5/T6/T7）都是可被静态复制或通用框架即可凑齐的表面特征。所以达标白名单只用确证级单独成立、或强证据跨表面联合，弱证据无论多少都不单独达标——见 判定规则。

资产哈希（assetHashMatch）

首页引用的 assets/index-<hash>.js / .css 文件名集合。哈希由 Vite 构建产物的内容算出，同版本恒定、跨版本不同。assetHashMatch = 该集合与指纹库某版本条目的 asset_hashes 精确相等（setsEqual，非子集）。它既是隐式版本反推的信号，也是判定规则 C3 的一个条件。fingerprints.go:41-56

WARNING · 资产哈希的采集面与 T5/T6 同源

资产哈希是从首页明文 HTML 正则抽取文件名得到的，扫描器不二次请求那两个资产、不校验真实字节。所以单独的资产哈希可被「镜像一份真首页」廉价伪造，与伪造 T6 同难度。因此 C3 要求它必须与 T2（动态 WS 行为）联合。详见 判定规则 C3。

scan_blocks 状态机（多机协同断点续扫）

progress 模块用 scan_blocks 表把大范围切成 CIDR 块、在多 worker 间分发，是这张表的权威落点。状态流转：src/agentsectool_scanner/progress/blocks.py

  • seed：按 /prefix 切块，INSERT OR IGNORE 登记为 pending（重复登记幂等）。
  • claim：worker 用 BEGIN IMMEDIATE 串行化，原子领取一个 pending 块、置 claimed 并记 worker 与领取时间——保证两个 worker 不会领到同一块。
  • mark_done：worker 扫完该块，置 done。
  • reset_stale：把领取超过 3600s 仍未完成的块退回 pending（回收僵死 worker 占用的块），供他人重领。这是运维定期动作，见 维护。

INVARIANT · 快照模型与时间窗口去重

一个 jsonl = 一个快照，snapshot_date 取该 jsonl 内 ts 的日期众数（跨天扫描以最多的日期为准，整个 jsonl 统一归此版）。同资产每个快照日期一行——既有快照不被覆盖，作镜像留存。store --in 入库时同一快照内同 asset_type:ip:port 以最新结果 REPLACE。前端按时间窗口加载：窗口内同 asset_type:ip:port 取最新 snapshot_date 那条（ROW_NUMBER() OVER (PARTITION BY asset_type,ip,port ORDER BY snapshot_date DESC)=1）。这同时解决两个问题：窗口内取并集 → 部分重扫不会让未扫目标假消失；太久没扫到的点落在窗口外 → 自动淘汰，不会永久残留。src/agentsectool_scanner/store/schema.sql; src/agentsectool_scanner/store/load.py:_snapshot_date; src/agentsectool_scanner/dashboard/server.py:_windowed_assets

安全与运营边界

授权边界

WARNING · 对公网的主动扫描属运营动作，须授权

ocprobe / masscan / zmap 会对目标 IP 发起真实连接。最小路径用本地靶机；大规模扫描须在获授权的主机上、对获授权的目标进行。本项目扫的 OpenClaw 实例多来自 ClawSec / FOFA 已公开列出的暴露实例。

默认允许 / 默认禁止行为（落到具体动作）

  • 允许（探测全程只读）：HTTP 升级请求、无鉴权 GET（/healthz、/、control-ui-config、/favicon.ico）、读 WS 首帧。
  • 禁止（有单测固化于函数级）：发任何 WS 数据帧、发 connect 帧或 config.apply（CVE 利用面）。ws.go 只 Write 一次升级请求；probe_test.go 断言不含越界帧
  • 可达性门槛：先一次 TCP 连接，连不上的目标直接返回、不发 HTTP/WS。probe.go

WARNING · 「整条链路只读」是函数级证据，非整链抓包

只读不变量由 probeWS 的函数级单测固化（断言其写入字节）。整条 runner 调用链的「只读」未做抓包断言。UNVERIFIED

速率与限制

  • -rate（默认 50）：单 IP 每秒发包上限，但实现为 per-worker time.Sleep、非跨 goroutine 全局限速。同一 IP 多端口入队后各 worker 互不协调。HTML 不声称「严格全局单 IP 速率」。main.go minInterval
  • masscan 限速靠 --rate；blocklist 默认排除 16 个 RFC 保留/私有网段，非法 IP 一律排除。src/agentsectool_scanner/discovery/blocklist.py:6-30

凭据 / 敏感数据 / 网络

  • 凭据存储：FOFA、MaxMind 和模型 API 凭据从本地 .ini 或环境变量读取，相应本地配置已由 Git 忽略。
  • 敏感数据：各模块的 data/ / output/ 目录、prober/output/、tools/scan_test/output/results/、各 .sqlite 含真实暴露主机与原始请求/响应，已 gitignore。probe_records 的响应原文做了二进制摘要清洗、前端 HTML 转义防 XSS。
  • 明文传输：ClawSec 平台走 http://clawsec.tcode.com.cn/api（明文，非 https）。请求/响应明文传输。
  • 看板：仅监听 127.0.0.1，不对公网暴露。

误操作与止损

  • make clean 删除各模块本地 data/ / output/ 目录，包含 SQLite 库和临时扫描产物，不可逆。运行前确认。
  • ocprobe -o / discovery 输出对同名文件截断重建（非追加），覆盖前确认。
  • 入库后历史在 observations（append-only），探测可安全重跑。
