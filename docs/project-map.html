AgentSecTool Scanner：项目地图

AgentSecTool Scanner 是面向多资产类型的测绘平台。平台包含候选发现、内置与动态探测、漏洞需求驱动的能力派生、Codex 智能体运行时、证据归档和结果展示；OpenClaw 是当前内置的资产类型。

项目定位
候选发现、资产探测、版本测定、证据归档、看板展示的综合测绘平台。
推荐探测入口
prober/bin/assetprobe --type openclaw
Python 包根
src/agentsectool_scanner/，使用 src-layout 隔离源码包和仓库杂项。
派生工作台
http://127.0.0.1:8787/derivation
智能体运行时
src/agentsectool_scanner/agent_runtime/，负责 Codex 执行、模型接入、审批和 trace。
OpenClaw 边界
OpenClaw 是内置资产类型，其判定、版本和证据采集均位于专属 detector 分支。

src-layout 说明

src/agentsectool_scanner 多一层包名是 Python 项目的标准 src-layout。这样可以避免从仓库根目录误导入同名脚本，也让 PYTHONPATH=src:. 下的包边界更清楚。

主链路

平台链路把外部候选和主动发现统一成候选集合，再交给资产类型 detector 只读探测。detector 输出统一摘要字段和专属证据，平台层负责入库、查询和展示。

候选来源

主动扫描、FOFA、ClawSec 或人工种子。

prober/targeting
detector 调度

按 --type 选择资产类型，并发、限速、超时。

prober/output/results.jsonl

Go 探测侧

Go 侧负责高并发探测和资产类型分发。平台层只认识 detector 接口和统一摘要；OpenClaw 的协议细节留在 OpenClaw 分支中。

路径职责重点文件 / 对象什么时候看这里
prober/cmd/assetprobe 平台化探测 CLI。 main.go、progress.go、config_load.go 改 --type、并发、限速、输出 JSONL、进度汇报、配置加载。
prober/detectors 资产类型注册表。 Detector、Register、New、Summary 新增资产类型、调整平台摘要字段、查看 detector 合约。
prober/detectors/openclaw OpenClaw detector 适配层。 Detector.Probe、DefaultPorts 把 OpenClaw 探测核心接入平台接口，或修改默认端口与指纹加载。
prober/detectors/dynamic 动态项目测试项执行器。 CompositeDetector、worker 池、能力索引校验 调整容器隔离、JSONL worker 协议、多测试项聚合或镜像 digest 校验。
prober/openclaw OpenClaw 判定、取版本、证据采集。 Probe、probeWS、probeHTTP、score、determineVersion 改 OpenClaw 特征、版本推断、证据结构、只读安全边界。
prober/targeting 目标展开与候选解析。 targets.go 新增候选输入格式，或修复 CIDR、通配地址、CSV 第一列解析。
prober/zgrab ZGrab2 OpenClaw 模块。 module.go、main.go 需要接入 ZGrab2 生态或复用其扫描输出格式时看这里。

Python 平台侧

Python 侧承担候选发现、能力派生、智能体运行、持久化、看板、进度队列和地理富化。具体项目测试逻辑保存在内置 detector 或已入仓能力包中；模型线程、容器权限、审批事件和 trace 由独立运行时模块管理。

路径职责重点文件 / 对象什么时候看这里
src/agentsectool_scanner/discovery 候选发现。 active.py、passive.py、merge.py、blocklist.py 调整 masscan/ZMap/internal 发现，或合并、去重、黑名单策略。
src/agentsectool_scanner/store 扫描结果入库与查询。 schema.sql、load.py、report.py 改 SQLite 表结构、快照规则、报告查询、统计口径。
src/agentsectool_scanner/dashboard 本地资产看板与派生工作台。 server.py、index.html、derivation.html 改扫描报告、派生任务交互、API、地图或展示模板渲染。
src/agentsectool_scanner/derivation 测绘能力派生与治理。 packages.py、agent.py、harness.py、capabilities.py 改需求包契约、会话工具、验收规则、容器策略、候选审核或能力索引。
src/agentsectool_scanner/agent_runtime Codex 智能体运行时。 codex_runtime.py、launcher.py、worker.py、trace/ 改模型提供方、Codex thread/turn、容器权限、审批、协议兼容或 trace。
src/agentsectool_scanner/progress 主动扫描块队列。 blocks.py、__main__.py 做断点续扫、分片领取、扫描轮次管理。
src/agentsectool_scanner/geoip 离线地理富化。 lookup.py、backfill.py、geoip.example.ini 接入 GeoLite2、补填国家/省市/经纬度、排查地图坐标。
src/agentsectool_scanner/paths.py Python CLI 默认路径。 SCANNER_DB、DERIVATION_DB、AGENT_RUNTIME_CONFIG 默认数据位置调整时先看这里，再同步 Makefile 与文档。

智能体运行时

智能体运行时为 Scanner 提供独立的 Codex 执行环境。宿主进程读取本地模型配置并生成去除密钥的容器配置，受限容器启动 Codex app-server，再通过 OpenAI 或 LiteLLM 的 Responses API 访问所选模型。thread、turn、工具事件、审批决定和模型协议记录写入任务 trace。

宿主入口

加载模型档案，准备 trace 路径和容器参数。

launcher.py
受限容器

只读根文件系统、临时工作区、资源上限和独立状态卷。

runtime.Dockerfile
Codex 执行

创建或续接 thread，执行 turn 并处理工具审批。

codex_runtime.py
模型服务

使用 OpenAI 直连或 LiteLLM 网关提供 Responses API。

config/ · compose.yaml
运行记录

保存流式事件、工具结果、协议载荷和行为图。

trace/ · render/

运行时主链路从宿主配置进入受限容器，经 Codex 调用模型服务；执行事件返回 trace，真实模型密钥不写入容器配置或运行记录。

路径职责关键约束
config/Pydantic 配置模型、模型档案和本地配置加载。提交的示例不含密钥；本地配置由 Git 忽略。
codex_runtime.pyCodex SDK 初始化、thread 续接、turn 执行和事件桥接。模型提供方与运行时分离；LiteLLM 工具序列经过协议归一化。
launcher.py、worker.py构建运行镜像、注入临时配置并启动容器内进程。密钥只通过进程环境传递，不进入命令参数和配置 JSON。
core/、trace/审批对象、运行结果、JSONL 事件和模型载荷记录。敏感请求头被遮蔽；原始载荷采集由配置控制。
doctor.py、tests/检查 SDK 版本、app-server、模型协议、工具循环和续接能力。模型配置通过协议检查后才适合承担长时间任务。
当前执行边界

derivation/agent.py 承载测绘能力派生的业务工具循环，agent_runtime/ 承载通用 Codex 执行环境。当前 DerivationService 仍实例化前者，两个模块之间没有运行时适配调用。

tools 地图

tools/ 放外部测绘源和一次性辅助工具。它们可以生产候选、指纹或统计，但不替代 detector 的最终确认。

tools/fofa

FOFA 拉取、增量查询、导出候选和省份版本统计。凭据走环境变量或本地配置模板，不入库。

候选源 输出 tools/fofa/output/candidates.csv。

tools/clawsec

ClawSec 快照拉取、长期暴露分析、与 FOFA 库做交集。

外部参考 第三方标注不作为本项目确认事实。

tools/scope

从 APNIC 等来源生成扫描范围，当前用于中国网段候选范围。

范围工具 输出默认在 tools/scope/output/。

tools/fingerprint

采集、迁移、构建 OpenClaw 指纹语料，用于补充 prober/fingerprints/openclaw.json。

离线工具 适合扩展版本覆盖率。

运行数据

默认运行数据按生产者就近存放。源码提交不应包含真实扫描库、JSONL 输出、第三方平台凭据或 GeoIP 商用/授权数据文件。

命令入口

本地验证默认不向公网发包。手动执行 Python 模块时，需要在仓库根目录设置 PYTHONPATH=src:.。

# 全量测试
make test

# 智能体运行时：安装依赖并执行独立测试
make agent-setup
make agent-test

# 创建被 Git 忽略的本地模型配置后检查运行时或进入交互界面
cp config/agent-runtime.example.toml config/agent-runtime.toml
make agent-doctor
make agent

# 本地最小链路：discovery(localhost) -> assetprobe -> store -> stats
make demo

# 构建并查看资产类型
go -C prober build -o bin/assetprobe ./cmd/assetprobe
prober/bin/assetprobe --list-types

# 手动跑最小链路
export PYTHONPATH=src:.
python3 -m agentsectool_scanner.discovery --cidr 127.0.0.0/30 --ports 18789 --backend internal --allow-reserved
prober/bin/assetprobe --type openclaw --fingerprints prober/fingerprints/openclaw.json \
  -o prober/output/results.jsonl src/agentsectool_scanner/discovery/output/candidates.csv
python3 -m agentsectool_scanner.store --in prober/output/results.jsonl --stats
python3 -m agentsectool_scanner.dashboard

资产看板位于 http://127.0.0.1:8787/，派生工作台位于 http://127.0.0.1:8787/derivation。主动扫描公网前必须配置授权范围、黑名单、限速和可识别出口。

边界与注意

安全边界

探测器默认只读，不应发送修改状态、写配置、登录尝试或提交敏感数据的请求。JSONL、SQLite 和 probe_records 可能包含目标响应原文，应按敏感运行数据处理。

第三方数据口径

FOFA、ClawSec、Shodan、ZoomEye 等外部测绘标签只能作为候选或参照。平台内的确认、版本和证据以本项目 detector 的实时探测输出为准。

智能体执行边界

智能体只在授权环境和受限容器中执行任务。提交的配置模板不含真实凭据；本地密钥、LiteLLM 配置、Codex 状态和 trace 均按敏感运行数据管理。智能体产生的候选内容需要经过 Harness 和人工审核，不能直接成为生产测绘能力。

文档边界

docs/archive/ 保存可供追溯的历史总结。当前功能说明位于 docs/项目文档-v2.html，代码地图位于 docs/project-map.html，能力派生的实施顺序与验收条件位于 docs/开发任务/测绘能力派生开发任务.html。
