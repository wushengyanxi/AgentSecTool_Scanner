AgentSecTool Scanner · 本轮实现说明（Round 1：扫描核心）

把 OpenClaw 扫描的核心跑通：全网能力的发现、可靠判定、尽力取版本

本轮目标 · TL;DR

这一轮只解决"扫描问题"：构建一条端到端管线——发现（能吃任意 CIDR 直至 0.0.0.0/0 的主动扫描，叠加 FOFA 被动种子）→ 判定（可靠确认是不是 OpenClaw）→ 取版本（疏防实例直读、已防实例隐式反推）→ 落地（JSONL + SQLite）。

构建的是"全网扫描能力"，不是现在就真打全 IPv4（那是需要专用出口 IP 与合规前置的运营动作）；本轮以本地 openclaw:local 容器 + 小网段验证正确性。版本→CVE 映射本轮不做。探测器用 Go + ZGrab2 自定义模块，结果落地 JSONL + SQLite，整条扫描器跑在本地宿主机。

02

架构与数据流

一条管线：两路发现并入候选队列，候选交给 Go/ZGrab2 探测器做只读判定与取版本，结果写 JSONL 再入 SQLite。发现与落地用 Python 便于迭代，探测器用 Go 复用 ZGrab2 自带的并发/超时/重试/结构化输出——这正是全网扫描需要的健壮性。

方框为管线阶段，箭头为数据流向，灰底框是候选队列，深色终端框是出口（结果入 SQLite）。最下方虚线条带是贯穿全程的运行约束：扫描器跑在本地宿主机、探测只读、对单一主机礼貌限速。两路发现并取以补齐反代后与扫不到的主机。

03

探测器：判定与取版本

探测器是本轮的核心：一个 Go 写的 ZGrab2 自定义模块，对每个目标只做只读探测，据此判定是否 OpenClaw 并尽力取版本。

每个目标只做两类只读动作：几个无鉴权 HTTP GET，加一次 WebSocket 升级读首帧。

判定按跨表面组合打分：WebSocket 一连上就收到的 connect.challenge（最强信号）、/healthz 的 {"ok":true,"status":"live"}、首页 <title>OpenClaw Control</title> 与 favicon.ico 的 md5、/__openclaw/control-ui-config.json 的 401＋特征 CSP。务实判定阈值取"favicon ∧ 标题 ∧ /healthz 体"（≈0.999），WS 挑战作抗伪装加权。每条信号的判别力、获取难度与组合阈值，已在 openclaw-fingerprint-phase0-and-stack.html §01 论证，这里不重复。

取版本分两条路。疏于防护的实例（auth=none）：control-ui-config 返回 200，直读 serverVersion（高置信）。已防护的实例：该接口 401、握手版本需配对、probe 模式也非旁路——只能隐式反推，用首页引用的 assets/index-<hash> 资产名、整串 CSP、响应头集合、协议版本去比对指纹库，得版本或版本区间（中置信）。指纹库由 harness 逐版本起容器记录签名而成，本轮种子 2026.5.17。

只读不变量是硬约束：探测器永不发 connect 帧、永不发 config.apply 或任何会改目标状态的帧；以单测固化"发送白名单只含 WS 升级与 GET"。识别一台主机不需要、也不允许去动它。

05

部署与安全

几条运行约束从一开始就具备，不延后——它们既是合规边界，也是"全网能力"能稳的前提。

  • 部署位置：整条扫描器（发现/探测/落地）作为本地宿主机进程运行；SQLite 就是工作目录下一个文件（gitignore），无独立 DB 服务、无 DB 容器。唯一的容器是被扫的 openclaw:local 靶机。迁到专用 Linux 扫描主机时再提供扫描器 Dockerfile（本轮不做）。
  • 只读不变量：探测器永不发 connect/config.apply/任何改状态帧；单测固化发送白名单。
  • 礼貌与合规：每 IP 并发/速率上限（远低于服务端 MAX_PREAUTH_CONNECTIONS_PER_IP=32）；黑名单/保留网段排除（masscan --excludefile）；扫描源可识别占位（rDNS/说明页/abuse）。这些是"全网能力"的前置，不延后。
  • 凭据安全：FOFA 等 API key 走 config/scanner.yaml（gitignore），运行时读取，绝不硬编码（CLAUDE.md §6）。
06

端到端验证

全部在本地完成，不向公网发包。先起靶机，再逐段验证，最后一键跑通全链。

# 1) 起靶机（被扫的 OpenClaw，已防护姿态）
docker run -d --name oc-fp -p 18789:18789 \
  -e OPENCLAW_GATEWAY_TOKEN=$(openssl rand -hex 32) \
  openclaw:local node dist/index.js gateway --bind lan --port 18789 --allow-unconfigured

# 2) M1 探测器：对靶机判定 + 取版本
echo "127.0.0.1,18789" | ./prober/bin/prober openclaw --port 18789 -o results.jsonl

# 3) M3 发现：小网段 + 合并去重（FOFA 有凭据则一并种子）
python -m discovery --cidr 127.0.0.0/30 --ports 18789 --backend masscan

# 4) M4 落地 + 全链
make load          # results.jsonl → SQLite，查 assets / observations 计数
make demo          # 发现(localhost) → 探测 → 落地，一键走通

本轮只求一件事：把 OpenClaw 扫描的核心立起来——发现具备全网能力、判定可靠、版本尽力可得、结果可落地，且全程只读、对单机礼貌。指纹与置信度的论证在 Phase 0 文档里已经做实，这一轮把它变成能跑的 ZGrab2 模块与 Python 发现/落地；版本对照 CVE、规模化存储与面板留给后续阶段。
