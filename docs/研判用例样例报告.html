AgentSecTool · OpenClaw 暴露面普查 · 研判用例样例

研判用例样例报告：七类情景的研判依据

Premise

本文档为研判机制（见《研判机制与版本取证》）的七类典型情景各取一个真实扫描用例。每例的呈现遵循同一结构：研判结论与版本取证置于顶部，其下逐个列出探针——每个探针只陈述自己这一步的观测事实（请求得到何种响应、该测试项是否命中），不在探针内重复证据级别或达标逻辑；研判规则如何由这些观测得出结论，统一收在每例末尾的"研判小结"里讲一次。所有请求/响应均为数据库真实落库原文。本页是逐主机报告的呈现模板。

用例 01

C1 命中，版本直读

rule=C1 · direct

control-ui-config 端点返回 200 且自报版本号，研判直接达标，版本亦从该响应读出，无须反推——研判与版本取证都最确定的一类。

研判结论
1.12.59.189:18789True · C1

命中测试项 T1 · T2 · T4 · T5 · T6。

版本取证
版本 2026.3.8（direct）：从 control-ui-config 端点直读服务端自报的 serverVersion。

用例 07

探不到：失败原因分类（不收录）

error_type · 不入库

既非 OpenClaw、又无任何测试项命中、且 HTTP 侧无响应的目标，会被补做一次 TCP 连接以区分失败原因。这类结果不收录进资产库，但其分类对普查的规模估计很重要——把"端口开着但非目标服务"与"主机不可达"分开。无有效响应可附。

研判结论
（示例目标）False · 探不到

无任何测试项命中，研判 False。此类不计入资产收录，仅用于普查的可达性统计。

四种失败分类

down　TCP 端口开着，但探不出任何 OpenClaw 证据——端口上跑着别的服务，非目标。
connection_refused　端口关闭，连接被拒。
timeout　连接超时——主机不响应，或被链路丢弃。
unreachable　网络不可达 / 无路由。

每个用例的请求/响应均取自 data/scanner/scan_results.sqlite 的真实落库原文（二进制响应折为摘要）。每个探针只陈述自身观测，研判规则（C1/C2 与证据级别）在研判机制文档中定义、在各例"研判小结"中应用，可逐条对照 prober/openclaw 源码复核。
